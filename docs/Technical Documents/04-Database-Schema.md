## 1. DATABASE TECHNOLOGY

| Item | Detail |
|---|---|
| **Database** | PostgreSQL |
| **ORM** | Drizzle ORM (`drizzle-orm/node-postgres`) |
| **Schema File** | `shared/schema.ts` |
| **Config File** | `drizzle.config.ts` |
| **Connection** | `pg.Pool` (max 10 connections, 30s idle timeout) |
| **Migrations** | `npx drizzle-kit push` |

### Connection Config
```typescript
const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  max: 10,
  idleTimeoutMillis: 30_000,
  connectionTimeoutMillis: 5_000,
});
export const db = drizzle(pool, { schema });
```

### Degraded Mode
Agar `DATABASE_URL` set nahi â€” server crash nahi karta, balki in-memory `degradedProjectStore` use karta hai. All other DB-backed features silently degrade.

### TABLE: `projects`
**Purpose:** Har workspace/sandbox ka master record

```sql
CREATE TABLE projects (
  id          SERIAL PRIMARY KEY,
  name        VARCHAR(255) NOT NULL,
  description TEXT,
  framework   VARCHAR(64),
  sandbox_path TEXT,                          -- e.g. /home/runner/.sandbox/my-app-1234
  status      VARCHAR(32) DEFAULT 'idle',     -- idle | running | completed | failed
  created_at  TIMESTAMPTZ DEFAULT NOW(),
  updated_at  TIMESTAMPTZ DEFAULT NOW()
);
```

| Column | Type | Required | Notes |
|---|---|---|---|
| `id` | serial | âś… auto | Primary key |
| `name` | varchar(255) | âś… | User-given project name |
| `description` | text | âťŚ | Optional description |
| `framework` | varchar(64) | âťŚ | e.g. "react", "nextjs", "express" |
| `sandbox_path` | text | âťŚ | Absolute path to project files |
| `status` | varchar(32) | âś… | `idle`, `running`, `completed`, `failed` |
| `created_at` | timestamptz | âś… auto | Creation timestamp |
| `updated_at` | timestamptz | âś… auto | Last update |

**Relationships:** Referenced by ALL other tables via `project_id`
**Cascade:** ON DELETE â†’ cascades to all child rows

### TABLE: `agent_runs`
**Purpose:** Har AI run ka high-level lifecycle record

```sql
CREATE TABLE agent_runs (
  id          VARCHAR(64) PRIMARY KEY,        -- UUID generated by chatOrchestrator
  project_id  INTEGER REFERENCES projects(id) ON DELETE CASCADE,
  goal        TEXT NOT NULL,                  -- User ka original prompt
  status      VARCHAR(32) DEFAULT 'running',  -- running | completed | failed | cancelled
  started_at  TIMESTAMPTZ DEFAULT NOW(),
  ended_at    TIMESTAMPTZ,                    -- NULL jab tak run active hai
  result      JSONB                           -- Final result object
);
```

| Column | Type | Notes |
|---|---|---|
| `id` | varchar(64) | UUID format, chatOrchestrator generate karta hai |
| `project_id` | integer FK | Parent project |
| `goal` | text | User ka exact input |
| `status` | varchar(32) | `running` â†’ `completed`/`failed`/`cancelled` |
| `started_at` | timestamptz | Run start time |
| `ended_at` | timestamptz | NULL = still active |
| `result` | jsonb | `{ success, summary, filesChanged, ... }` |

### TABLE: `chat_messages`

CREATE TABLE chat_messages (

### TABLE: `tool_executions`
**Purpose:** Har tool call ka detailed, queryable record (structured, not flat)

```sql
CREATE TABLE tool_executions (
  id            SERIAL PRIMARY KEY,
  execution_id  VARCHAR(64) NOT NULL UNIQUE,   -- UUID per execution
  run_id        VARCHAR(64) REFERENCES agent_runs(id) ON DELETE CASCADE,
  project_id    INTEGER REFERENCES projects(id) ON DELETE CASCADE,
  step_index    INTEGER,                        -- Orchestration step number
  tool_name     VARCHAR(128) NOT NULL,          -- e.g. "write_file"
  tool_category VARCHAR(64),                    -- filesystem | coding | terminal | ...
  status        VARCHAR(32) NOT NULL DEFAULT 'running',  -- running | success | failed
  args_json     JSONB,                          -- Tool input arguments
  result_json   JSONB,                          -- Tool output
  error_text    TEXT,                           -- Error message agar failed
  duration_ms   INTEGER,                        -- Execution time
  retry_count   INTEGER NOT NULL DEFAULT 0,
  replay_safe   BOOLEAN NOT NULL DEFAULT true,  -- Kya retry safe hai?
  started_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  ended_at      TIMESTAMPTZ
);

-- Indexes for fast queries
CREATE INDEX tool_executions_run_id_idx      ON tool_executions(run_id);
CREATE INDEX tool_executions_project_id_idx  ON tool_executions(project_id);
CREATE INDEX tool_executions_tool_name_idx   ON tool_executions(tool_name);
CREATE INDEX tool_executions_started_at_idx  ON tool_executions(started_at);
```

### TABLE: `checkpoints`
**Purpose:** Run ke baad complete file state snapshot

```sql
CREATE TABLE checkpoints (
  id              SERIAL PRIMARY KEY,
  checkpoint_id   VARCHAR(64) NOT NULL UNIQUE,  -- UUID
  project_id      INTEGER REFERENCES projects(id) ON DELETE CASCADE NOT NULL,
  run_id          VARCHAR(64) REFERENCES agent_runs(id) ON DELETE SET NULL,
  trigger         VARCHAR(32) NOT NULL,          -- auto | manual
  status          VARCHAR(32) NOT NULL DEFAULT 'stable',  -- stable | rolled-back
  git_commit_sha  VARCHAR(64),                   -- Git SHA agar available
  file_count      INTEGER NOT NULL DEFAULT 0,
  label           TEXT,                          -- Human-readable name
  description     TEXT,
  created_files   JSONB DEFAULT '[]',            -- ["src/App.tsx", ...]
  modified_files  JSONB DEFAULT '[]',
  deleted_files   JSONB DEFAULT '[]',
  file_snapshots  JSONB DEFAULT '{}',            -- { "src/App.tsx": "content..." }
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX checkpoints_project_id_idx ON checkpoints(project_id);
CREATE INDEX checkpoints_run_id_idx     ON checkpoints(run_id);
CREATE INDEX checkpoints_created_at_idx ON checkpoints(created_at);
```

**`file_snapshots` Structure:**
```json
{
  "src/App.tsx": "import React from 'react'...",
  "src/index.css": "body { margin: 0; }",
  "package.json": "{ \"name\": \"my-app\" ... }"
}
```

### DEPLOYMENT TABLES (5 tables)
**Purpose:** App deployment ka complete lifecycle

```sql
-- Main deployment record
CREATE TABLE deployments (
  id           SERIAL PRIMARY KEY,
  project_id   INTEGER REFERENCES projects(id) ON DELETE CASCADE NOT NULL,
  status       VARCHAR(32) NOT NULL DEFAULT 'idle',  -- idle | building | live | failed
  url          TEXT,                                  -- Live URL
  region       VARCHAR(64) NOT NULL DEFAULT 'us-east-1',
  environment  VARCHAR(32) NOT NULL DEFAULT 'production',
  steps        JSONB NOT NULL DEFAULT '[]',           -- Deployment step log
  error        TEXT,
  started_at   TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ,
  updated_at   TIMESTAMPTZ DEFAULT NOW()
);

-- Custom domains
CREATE TABLE deployment_domains (
  id         SERIAL PRIMARY KEY,
  project_id INTEGER REFERENCES projects(id) ON DELETE CASCADE NOT NULL,
  name       VARCHAR(255) NOT NULL,   -- e.g. "myapp.com"
  status     VARCHAR(32) NOT NULL DEFAULT 'pending',
  added_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Encrypted secrets
CREATE TABLE deployment_secrets (
  id              SERIAL PRIMARY KEY,
  project_id      INTEGER REFERENCES projects(id) ON DELETE CASCADE NOT NULL,
  key             VARCHAR(255) NOT NULL,
  encrypted_value TEXT NOT NULL,         -- AES encrypted
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- App configuration
CREATE TABLE deployment_settings (
  id          SERIAL PRIMARY KEY,
  project_id  INTEGER REFERENCES projects(id) ON DELETE CASCADE NOT NULL,
  app_name    VARCHAR(255) NOT NULL DEFAULT 'my-app',
  environment VARCHAR(32) NOT NULL DEFAULT 'production',
  region      VARCHAR(64) NOT NULL DEFAULT 'us-east-1',
  is_public   BOOLEAN NOT NULL DEFAULT true,
  db_url      TEXT,                    -- Production DB URL
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Auth configuration
CREATE TABLE deployment_auth_config (
  id                  SERIAL PRIMARY KEY,
  project_id          INTEGER REFERENCES projects(id) ON DELETE CASCADE NOT NULL,
  providers           JSONB NOT NULL DEFAULT '[]',   -- ["google", "github"]
  require_email_verif BOOLEAN NOT NULL DEFAULT true,
  session_expiry      VARCHAR(16) NOT NULL DEFAULT '7d',
  updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## 4. CASCADE BEHAVIOR

| Parent Delete | Child Table | Behavior |
|---|---|---|
| `projects` deleted | `agent_runs` | CASCADE delete |
| `projects` deleted | `chat_messages` | CASCADE delete |
| `projects` deleted | `tool_executions` | CASCADE delete |
| `projects` deleted | `checkpoints` | CASCADE delete |
| `projects` deleted | `deployments` | CASCADE delete |

| `agent_runs` deleted | `checkpoints` | SET NULL (runId) |
