AGENT LIFECYCLE INDICATOR SPECIFICATION

OBJECTIVE

Implement a unified agent lifecycle indicator system.

The animated central orb/logo indicator must remain active while a run is executing and transition through lifecycle states based on backend events.

Do not invent lifecycle states.

Use only actual backend lifecycle events.

==================================================
INDICATOR RULES

Default State:

Idle

When no active run exists:

show Idle state.

==================================================
THINKING

Trigger:

chat.run.started

OR

runAgent()

OR

isAgentThinking(true)

Backend Source:

chatOrchestrator.startRun()

Frontend State:

Thinking

UI:

Show animated orb.

Label:

"Thinking..."

Description:

"Analyzing request"

Exit When:

First planning event received.

==================================================
PLANNING

Trigger:

orchestration.workflow.started

Backend Source:

orchestration-loop.ts

Frontend State:

Planning

UI:

Animated orb remains active.

Label:

"Planning..."

Description:

"Building execution plan"

Exit When:

First phase execution starts.

==================================================
DELEGATING

Trigger:

phase.started

AND

agent assignment detected

Backend Source:

orchestration phase dispatch

Frontend State:

Delegating

UI:

Animated orb remains active.

Label:

"Delegating..."

Description:

"Assigning work to agents"

Exit When:

First tool execution starts.

==================================================
WORKING

Trigger:

agent.tool_call

status = running

Backend Source:

tool execution

Frontend State:

Working

UI:

Animated orb remains active.

Label:

"Working..."

Description:

Current tool label

Examples:

Reading Files
Searching Project
Generating Component
Executing Command

Exit When:

Tool completes.

==================================================
ANIMATION RULE

The animated orb/logo indicator (center icon) must remain continuously animated during:

Thinking
Planning
Delegating
Working
Writing
Editing
Testing
Verifying
Deploying

Animation stops only when:

Completed
Failed
Cancelled

==================================================
STATE PRIORITY

Highest Priority:

Failed
Cancelled
Completed

Then:

Deploying
Verifying
Testing
Writing
Working
Delegating
Planning
Thinking

Only one active lifecycle state may be displayed at a time.

Latest valid lifecycle event wins.

==================================================
END RESULT

Backend Event
↓
SSE/WebSocket
↓
Frontend Event Handler
↓
Lifecycle State
↓
Animated Orb Indicator
↓
Status Label
↓
Status Description

The UI must always reflect the current backend execution phase in real time.
